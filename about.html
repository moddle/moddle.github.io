Overview

In short, Moddle is a Minecraft launcher that let's you create, play, and distribute your very own modpacks quickly and easily. It is a project to revolutionize modpack building as we know it today. We have two equally-important goals for this project. The first is to build the best possible modpack-creation platform: A launcher that offers incomparable ease-of-use, but is more powerful than any modpack platform ever before. Our second goal is to be 100% community-oriented. As members of the Minecraft and modding communities, we wanted to build "a modpack platform by the players, for the players". With that goal in mind, it was a given that Moddle be completely Open Source. As such, our complete codebase is available on GitHub in our project repository, licensed under the MIT license.

How it Works

Moddle is built around the core concept of the Universal Mod Cache. The Universal Mod Cache (or just 'Cache' for short) is a folder in which the launcher stores all of the mods necessary to build its modpacks. This means that modpack creators no longer need to collect and package each mod themselves into a large distributable archive along with their config files. Instead, they need only provide a Moddle-compatible config file listing the mods to install and any other configuration options for the modpack.

Another one of the more powerful features of Moddle is auto-configuration. When using mods provided in the Cache, the launcher can automatically set up each mod's configuration files to cooperate with the other mods that are being installed. This auto-configuration system also handles block and item IDs, which means that you will be able to safely transfer worlds from one modpack to another!

It is important to note that, due to our early development stage (See dev status section below), we have not yet implemented a user interface for building modpacks. This is not to say that building a modpack will be hard as a result; creating modpacks is an easy process as it is now. You can learn how to build packs by watching our tutorial videos.

Permissions

Everyone who has worked with modpacks before knows that mod permissions are a more-than-important requirement to creating your modpack. Although Moddle is not a "modpack" technically speaking, we do not include any mods in the Cache that do not give permission for their use in public modpacks. You can include these mods in your own private packs by adding them to the mini-caches of those packs, but they will not and cannot be used in modpacks that we distribute as default modpacks or ones that can be classified as a "public modpack". Automated permissions validation was included in older versions of the launcher. This feature is not yet built into the newest versions, but we will be working on it.

Development Status

The Moddle launcher is still under full development. Shortly after starting work on the project, however, we decided to distribute 'pre-release' versions of the launcher. Having pre-1.0 versions used actively will help us get player feedback early, find nasty bugs, and get suggestions on how to improve the application. It will drive us towards our goal of community-integration in ways that we could not achieve with a generic release schedule.

You can help us by getting Moddle and trying it out for yourself! Report crashes and bugs that you encounter on our GitHub issue tracker. Feedback and suggestions can also be submitted on the issue tracker. Tell us what your impressions are, what features you want, or even how you think the launcher should look. We need your opinions to make this the best launcher that it can possibly be!
